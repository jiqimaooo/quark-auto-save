## 与原项目的区别

本仓库在原项目基础上做了以下改进和调整：

### 插件系统优化

- **插件测试按钮**：系统配置页每个插件旁边新增「测试」按钮，配完参数可即时验证连通性，无需跑完整任务
- **插件间延时执行**：每个插件支持配置 `delay`（秒），解决 alist 刷新未完成 Emby 就去扫描的问题。例如：alist 刷新目录 → 等 5 秒 → Emby 扫描媒体库
- **任务级插件开关**：补全了 alist、plex、smartstrm、alist_strm 等插件的任务级 `enable` 控制，所有插件在任务选项中统一可见、可开关
- **仅展示已激活的插件**：未在全局配置中填参数的插件不会出现在任务选项里，减少干扰

### Emby 插件重构

- 删除了原有的「按任务名搜索匹配剧集」逻辑（任务名和 Emby 库内名称很难完全一致）
- 改为两种可选模式：
  - **全库扫描**（默认）：`POST /Library/Refresh`，简单可靠
  - **单剧刷新**：勾选 `item_mode` 并手动填入 Emby 剧集 ID，精确快速

### 前端 UI 调整

- 任务插件选项 JSON 编辑器允许手动添加字段（`:plus="true"`）
- 系统配置页卡片式布局优化
- 删除确认交互优化

### Bug 修复

- 修复 `auto_unarchive` 插件未配置时初始化 `AttributeError` 的问题
