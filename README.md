## 与原项目的区别

本仓库在原项目基础上做了以下改进：

### 插件系统重构

- **两层独立控制**：每个插件有两层开关——
  - 第一层：系统配置页填好参数（url/token等），插件与目标服务连通后自动激活（`is_active`）
  - 第二层：填好参数不代表要用，需手动打开 `enable` 开关，插件才会在任务中**展示**和**执行**
  - 可以随时关闭开关，不需要删除配置参数，灵活切换

- **仅展示已启用插件**：任务列表的插件选项中，只有「填了参数 + 打开了开关」的插件才会出现，不会看到一堆没配过的插件

- **插件测试按钮**：系统配置页每个插件旁边有「测试」按钮，配完参数点一下即时验证连通性，不用跑完整任务才知道通不通

- **插件间延时执行**：每个插件支持配置 `delay`（秒），解决 alist 刷新未完成 Emby 就去扫描导致扫不到新数据的问题。例如：`alist 刷新目录 → 等 5 秒 → Emby 扫描媒体库`

- **补全任务级开关**：alist、plex、smartstrm、alist_strm 等原本不支持任务级控制的插件，现在统一支持 `enable` 开关

### Emby 插件重构

- 删除了原有的「按任务名搜索匹配剧集」逻辑——任务名和 Emby 库内名称几乎不可能完全一致，匹配不上的概率极高
- 改为两种可选模式，`item_mode` 勾选框控制：
  - **不勾选（默认）= 全库扫描**：`POST /Library/Refresh`，简单可靠，不管什么任务名都能刷到
  - **勾选 = 单剧刷新**：手动填入 Emby 剧集 ID（`media_id`），精确快速，只刷新一部剧

### 前端 UI 调整

- 系统配置页卡片式布局
- 任务插件选项 JSON 编辑器可手动添加字段
- 删除确认交互优化

### Bug 修复

- 修复 `auto_unarchive` 插件未配置时初始化 `AttributeError` 的问题
- 修复 `/data` 接口返回的 `task_plugins_config` 为启动时缓存值，修改配置后不即时生效的问题（改为每次请求动态计算）
